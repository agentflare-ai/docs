## Overview

Agentflare is a hosted proxy service - there's no software to install. Instead, you'll:

1. Create an Agentflare account
2. Add your tool servers via the dashboard
3. Get proxy URLs for each server
4. Configure your AI agents to use the proxy

Before getting started, ensure you have:

## Step 1: Create Your Account

<Steps>
  <Step title="Sign Up">
    Navigate to [app.agentflare.com](https://app.agentflare.com/login) and create an account
  </Step>
  <Step title="Verify Email">
    Check your email and verify your account
  </Step>
  <Step title="Create Workspace">
    Set up your first workspace (e.g., "Production", "Development")
  </Step>
  <Step title="Choose Plan">
    Select a plan that fits your needs (Free tier available)
  </Step>
</Steps>

## Step 2: Get Your API Key

<Steps>
  <Step title="Navigate to API Keys">
    Go to **Dashboard ‚Üí Settings ‚Üí API Keys**
  </Step>
  <Step title="Create New Key">
    Click "Create API Key" and give it a descriptive name
  </Step>
  <Step title="Copy the Key">
    Save the API key securely - you won't be able to see it again
  </Step>
  <Step title="Set Environment Variable">
    Add the key to your environment:
    ```bash
    export AGENTFLARE_API_KEY="your_api_key_here"
    ```
  </Step>
</Steps>

<Warning>
  Never commit API keys to version control. Always use environment variables or secure secret management.
</Warning>

## Step 3: Add Your Tool Server

Navigate to **Dashboard ‚Üí Tool Servers ‚Üí Add Server** and configure your tool server:

<Tabs>
  <Tab title="HTTP/HTTPS Server">
    For HTTP/HTTPS based MCP servers:

    ```json
    {
      "name": "search-server",
      "description": "Search and retrieval tools",
      "type": "http",
      "endpoint": "https://your-server.com/mcp",
      "authentication": {
        "type": "bearer",
        "token": "your-server-token"
      },
      "timeout": 30000
    }
    ```

    <AccordionGroup>
      <Accordion title="Authentication Types">
        - **Bearer Token**: `{"type": "bearer", "token": "..."}`
        - **API Key**: `{"type": "api_key", "key": "...", "header": "X-API-Key"}`
        - **Basic Auth**: `{"type": "basic", "username": "...", "password": "..."}`
        - **OAuth2**: `{"type": "oauth2", "client_id": "...", "client_secret": "..."}`
      </Accordion>
    </AccordionGroup>
  </Tab>

  <Tab title="WebSocket Server">
    For WebSocket based MCP servers:

    ```json
    {
      "name": "realtime-server",
      "description": "Real-time collaboration tools",
      "type": "websocket",
      "endpoint": "wss://your-server.com/mcp",
      "authentication": {
        "type": "bearer",
        "token": "your-server-token"
      },
      "heartbeat": {
        "enabled": true,
        "interval": 30000
      }
    }
    ```
  </Tab>

  <Tab title="Stdio Server">
    For locally running stdio servers:

    ```json
    {
      "name": "local-server",
      "description": "Local development server",
      "type": "stdio",
      "command": "/usr/local/bin/mcp-server",
      "args": ["--config", "/etc/mcp/config.json"],
      "env": {
        "MCP_SERVER_TOKEN": "your-token",
        "LOG_LEVEL": "info"
      },
      "working_directory": "/app"
    }
    ```

    <Note>
      Stdio servers run on Agentflare's infrastructure. Ensure your server binary is accessible or use HTTP/WebSocket for external servers.
    </Note>
  </Tab>
</Tabs>

## Step 4: Get Your Proxy URL

After adding your tool server, Agentflare generates a unique proxy URL:

```
https://<server-slug>.proxy.agentflare.com
```

**Example:**
```
https://search-server.proxy.agentflare.com
```

Copy this URL - you'll use it to configure your AI agents.

## Step 5: Configure Your Agent

Update your AI agent or MCP client to use the Agentflare proxy:

<CodeGroup>

```typescript TypeScript / JavaScript
import { Client } from "@modelcontextprotocol/sdk/client/index.js";
import { HTTPTransport } from "@modelcontextprotocol/sdk/client/http.js";

const client = new Client({
  transport: new HTTPTransport({
    baseUrl: process.env.AGENTFLARE_PROXY_URL,
    headers: {
      "Authorization": `Bearer ${process.env.AGENTFLARE_API_KEY}`
    }
  })
});

await client.connect();
```

```python Python
from mcp import Client
from mcp.transport import HTTPTransport
import os

client = Client(
    transport=HTTPTransport(
        base_url=os.getenv("AGENTFLARE_PROXY_URL"),
        headers={
            "Authorization": f"Bearer {os.getenv('AGENTFLARE_API_KEY')}"
        }
    )
)

await client.connect()
```

```go Go
import (
    "github.com/modelcontextprotocol/sdk-go/client"
    "github.com/modelcontextprotocol/sdk-go/transport"
)

client := client.New(
    transport.NewHTTP(
        os.Getenv("AGENTFLARE_PROXY_URL"),
        transport.WithHeaders(map[string]string{
            "Authorization": "Bearer " + os.Getenv("AGENTFLARE_API_KEY"),
        }),
    ),
)
```

</CodeGroup>

## Environment Configuration

### Development Environment

Create a `.env` file in your project root:

```bash .env
# Agentflare Configuration
AGENTFLARE_API_KEY=your_api_key_here
AGENTFLARE_PROXY_URL=https://your-server.proxy.agentflare.com

# Optional: Enable debug logging
AGENTFLARE_DEBUG=true
```

### Production Environment

For production, use your environment's secret management:

<Tabs>
  <Tab title="Docker">
    ```dockerfile
    FROM node:18-alpine

    WORKDIR /app

    # Install dependencies
    COPY package*.json ./
    RUN npm ci --only=production

    # Copy application
    COPY . .

    # Environment variables passed at runtime
    ENV NODE_ENV=production

    CMD ["node", "index.js"]
    ```

    ```bash
    docker run \
      -e AGENTFLARE_API_KEY="your_key" \
      -e AGENTFLARE_PROXY_URL="your_proxy_url" \
      your-agent-image
    ```
  </Tab>

  <Tab title="Kubernetes">
    ```yaml
    apiVersion: v1
    kind: Secret
    metadata:
      name: agentflare-secrets
    type: Opaque
    stringData:
      AGENTFLARE_API_KEY: "your_api_key_here"
      AGENTFLARE_PROXY_URL: "https://server.proxy.agentflare.com"
    ---
    apiVersion: apps/v1
    kind: Deployment
    metadata:
      name: ai-agent
    spec:
      template:
        spec:
          containers:
          - name: agent
            image: your-agent:latest
            envFrom:
            - secretRef:
                name: agentflare-secrets
    ```
  </Tab>

  <Tab title="AWS Lambda">
    ```javascript

      })
    });

    exports.handler = async (event) => {
      await client.connect();
      // Your Lambda logic
      return response;
    };
    ```

    Set environment variables in AWS Lambda console or via IaC tools.
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Authentication Errors (401)">
    **Symptoms**: Getting "Unauthorized" errors

    **Solutions**:
    - Verify API key is correct and not expired
    - Check the Authorization header format: `Bearer YOUR_KEY`
    - Ensure environment variables are properly loaded
    - Confirm API key has necessary permissions
  </Accordion>

  <Accordion title="Connection Timeouts">
    **Symptoms**: Requests timing out or failing to connect

    **Solutions**:
    - Check network connectivity and firewall rules
    - Verify proxy URL is correct
    - Ensure tool server is running and accessible
    - Check Agentflare status page for outages
  </Accordion>

  <Accordion title="Tool Server Not Found (404)">
    **Symptoms**: "Server not found" errors

    **Solutions**:
    - Verify the server-slug in your proxy URL
    - Check server is enabled in dashboard
    - Confirm workspace-slug is correct
    - Ensure server configuration is complete
  </Accordion>

  <Accordion title="Data Not Appearing">
    **Symptoms**: Tool calls work but don't show in dashboard

    **Solutions**:
    - Verify proxy URL is being used (not direct connection)
    - Check API key is included in requests
    - Look for errors in application logs
    - Wait a few seconds for data to appear (buffering)
  </Accordion>
</AccordionGroup>

## Platform-Specific Notes

<Tabs>
  <Tab title="Node.js">
    **Minimum Version**: Node.js 18+

    **Package Manager**: Use npm, yarn, or pnpm

    **Environment Variables**: Use `dotenv` for local development

    ```bash
    npm install dotenv
    ```

    ```javascript
    require('dotenv').config();
    ```
  </Tab>

  <Tab title="Python">
    **Minimum Version**: Python 3.8+

    **Virtual Environment**: Recommended

    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```

    **Environment Variables**: Use `python-dotenv`

    ```bash
    pip install python-dotenv
    ```
  </Tab>

  <Tab title="Go">
    **Minimum Version**: Go 1.19+

    **Modules**: Enable Go modules

    ```bash
    go mod init your-project
    ```

    **Environment Variables**: Use `godotenv` or built-in `os.Getenv`
  </Tab>
</Tabs>
